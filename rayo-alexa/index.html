Rayo Alexa skill

Product

Rayo Alexa skill enables users to listen to live radio and catchup shows provided by Bauer Media on the Alexa smart speakers.

Role

Research, product design, testing

Overview

As the first designer to ever join the Voice Connected Device (VCD) team, where the product had previously been developed solely by product managers and engineers without dedicated design support. I was brought in to establish a design foundation and integrate a structured design process into the existing workflow.

Problem

All user journeys were undocumented, existing features are not intuiative. All designs for screen devices needed to be updated with the New Rayo branding (previously Planet Radio).

Goal

Introduce a design process in the existing workflow. Document and improve all existing journeys, design new assets and UI for screen devices.

Mapping user journeys

This is the first time I got hands on Alexa devices, an Echo Dot (without screen), and an Echo Show (with screen). I documented every step starting from unboxing the devices all the way to finishing my account set up.

I wanted to document how far a first time user would interact with the device, and how much context can Alexa understand from me naturally speaking to it. Without given any information from the team, I tried different utterance to ask for different pieces of content.

Disappointing AI assistant

After testing the devices, it is clear to me that Rayo skill (or any other custom skill) were built with a binary approach, similar to a customer service chatbot. You have follow the exact command structure in order for Alexa to understand and perform the action.

Learning the utterance structure

There are 2 types of basic (but absolute) structure command that you must follow word by word in order to initate a skill to achieve your goal:

Open skill

"Open [skill name]", wait for skill to initiate then say the action you want it to perform.

One shot

"Ask [skill name] to [perform an action]".

Interaction model

I audited and documented all available Alexa content from a JSON file, and cataloged all synonyms that have been added for requesting specific shows.

Build intents with Voiceflow

Intent is equivalent to a feature in a mobile app. I create a production file where I build all existing intents we have inside to serve as a single source of truth.

They were built very close to the actual Alexa skill taking into account different conditions, synonyms, error feedback. They were also used as prototypes for user testing.

Screen device UI

I set up a new design system and built or redesign every page in a production file as a single source of truth.

All designs were built with the limitations of the platform, I worked with the principle developer to find the best layout possible for each tasks.

Account linking improvements

There are too many account linking methods, our support page alone has six separate articles explaining them. This creates high maintenance overhead for developers, when in reality, we only need one reliable solution.

Out of all the methods, link via the Alexa app is the only one that's bug free. I designed all the journeys, no matter where the start point is, from iOS or Android, from Legacy brand apps or the new Rayo app, from scanning QR code or initiate account link with Alexa, they all deeplinks you to the Alexa app to complete the account linking journey.

In-situ help

Many users struggle to account link, I introduce in-situ help feature to guide the users with support page on screen, and as well as voice guidance.

Continue listening

We introduced the "Continue Listening" feature as a benefit of linking your Rayo account. When users play content they've previously started, Alexa now prompts them to resume from where they left off.
